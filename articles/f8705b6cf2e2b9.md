# はじめに

この資料は、AWSをこれから始める社内メンバー向けの入門ガイドです。
AWSのアカウントを安全に運用することを目的としています。

# ルートユーザーログイン

:::message alert
ルートユーザーでのログインは初回セットアップ時のみ行い、通常の運用では使用しないでください。
:::


①ルートユーザーを選択し、ルートユーザーのメールアドレスを入力、
![ルートユーザーでログイン](/images/f8705b6cf2e2b9/image01.png =400x)


②ルートユーザーのパスワードを入力し、サインインします。
![ルートユーザーでログイン](/images/f8705b6cf2e2b9/image02.png =400x)

③AWSマネジメントコンソールが表示され、初回ログインできました
![AWSマネジメントコンソール](/images/f8705b6cf2e2b9/image03.png =400x)



# 即時で対策すべき設定

アカウント作成後、数日以内に実施すべき重要な設定です。これらは不正利用を防ぐための必須の設定となります。

## 1. ルートユーザーのMFA認証設定

ルートアカウントの安全性を高めるため、必ず多要素認証（MFA）を設定します。

:::message alert
ルートユーザーは最も強力な権限を持つため、MFA設定は必須です。
:::


①AWSマネジメントコンソールの右上端のAWSアカウント名のトグルを選択し、セキュリティ認証情報をクリックします。
![セキュリティ認証情報](/images/f8705b6cf2e2b9/image04.png =400x)

②IAMのセキュリティ認証情報の画面に遷移するので、MFA を割り当てるをクリックします
![MFAを割り当てる](/images/f8705b6cf2e2b9/image05.png =400x)

③デバイス名とMFAデバイスの種類を選択します。
![MFAデバイスの種類](/images/f8705b6cf2e2b9/image06.png =400x)

④QRコードを表示をクリックして、MFA設定用のQAコードを表示させ認証アプリ側でスキャンします。スキャン後認証アプリ上で表示される認証コードを入力し、MFAを登録します。
![MFA設定](/images/f8705b6cf2e2b9/image07.png =400x)

⑤MFAデバイスが登録されていることを確認します。
![MFAデバイスが登録されていることを確認](/images/f8705b6cf2e2b9/image08.png =400x)

## 2. IAMパスワードポリシーの設定

組織全体のIAMユーザーに対するパスワードポリシーを設定します。

①IAM > アカウント設定から、パスワードポリシーを変更します。
![パスワードポリシー](/images/f8705b6cf2e2b9/image09.png =400x)

- パスワードの最小文字数：8文字以上
- 大文字小文字の使用必須
- 数字の使用必須
- 記号の使用必須

②上記に加え、ユーザーにパスワードの変更を許可を追加しています。
![パスワードポリシー](/images/f8705b6cf2e2b9/image10.png =500x)

③変更を保存しようとすると、確認画面が表示されるのでそのまま設定します。
![確認画面](/images/f8705b6cf2e2b9/image11.png =500x)

④パスワードポリシー変更を確認します。
![パスワードポリシー変更確認](/images/f8705b6cf2e2b9/image12.png =500x)


## 3. IAMユーザーによる請求情報へのアクセス有効化

:::message
管理者用IAMユーザーが請求情報を確認できるようにすることで、日常的な費用管理が可能になります。
:::

①AWSマネジメントコンソールの右上端のAWSアカウント名のトグルを選択し、アカウントをクリックします。
![アカウント](/images/f8705b6cf2e2b9/image09.png =400x)

②IAM ユーザーおよびロールによる請求情報へのアクセス欄の編集を選択、そのままIAM アクセスをアクティブ化を有効化し、更新してください。
![IAM アクセスをアクティブ化](/images/f8705b6cf2e2b9/image10.png =400x)

③IAM ユーザー/ロールによる請求情報へのアクセスが有効化済みとなっていれば設定完了です。
![IAM ユーザー/ロールによる請求情報へのアクセスが有効化済み](/images/f8705b6cf2e2b9/image11.png =400x)  

## 4. 管理者用IAMユーザーの作成
リスクを回避するためにルートユーザーは可能な限り使わない方がよいです。代わりに管理者(Administrator)権限を持ったIAMユーザーを作成し、普段はその認証情報を使用してAWSアカウントを運用すべきです。

①まずはIAM > ユーザーグループからグループを作成します。
![グループ作成](/images/f8705b6cf2e2b9/image15.png =500x)

②任意のユーザーグループ名を設定し、許可ポリシーはAdministratorAccessを選択します。
![グループ作成](/images/f8705b6cf2e2b9/image16.png =500x)
![グループ作成](/images/f8705b6cf2e2b9/image17.png =500x)

③グループが作成されたことを確認します。
![グループ作成](/images/f8705b6cf2e2b9/image18.png =500x)

④IAM > ユーザーからユーザーを作成します。
![ユーザー作成](/images/f8705b6cf2e2b9/image19.png =500x)

⑤ユーザー名を設定し、グループにAdministratorsを選択します。
:::message alert
パスワードは初回ログイン後に再設定するオプションにしています。
:::
![ユーザー作成](/images/f8705b6cf2e2b9/image20.png =500x)


⑥ユーザーをグループに追加をクリックし、追加対象のユーザーグループを選択します。
![ユーザーをグループに追加](/images/f8705b6cf2e2b9/image21.png =500x)

⑦ユーザー作成画面が表示されるので、そのままユーザーを作成します。
![ユーザー作成確認](/images/f8705b6cf2e2b9/image22.png =500x)

⑧IAMユーザーの作成が完了すると、ユーザー名/パスワード等のコンソールサインインの情報が表示されます。
:::message alert
初期パスワードは画面遷移すると見られなくなるため、このページで忘れずに.csvファイル形式でダウンロードしておきましょう。
:::
![ユーザー作成確認](/images/f8705b6cf2e2b9/image23.png =500x)

⑨ルートアカウントからログアウトし、作成した管理者用IAMユーザーでAWSマネジメントコンソールにログインしてください。ダウンロードした.csvファイル内にあるコンソールサインインURL(例: `https://123456789012.signin.aws.amazon.com/console`)からログインポータルにアクセスする方法が一番楽です。

ユーザー/パスワード認証に成功すると、設定した通りパスワード変更が求められるため、任意のパスワードに変更します。
![パスワード変更](/images/f8705b6cf2e2b9/image24.png =500x)

⑩パスワード変更後、再度マネジメントコンソールへログインできるところまで確認できれば完了です。
![ログイン確認](/images/f8705b6cf2e2b9/image25.png =500x)

:::message alert
今後はルートユーザーにログインする必要はないため、以降は管理者用IAMユーザーでログインして実施してください。
:::

## 5. 管理者用IAMユーザー MFA認証の設定
①管理者用IAMユーザーについても、MFA認証の設定を必ずやっておきましょう。

IAMユーザーのMFA設定手順は、IAM > ユーザー > [IAMユーザー名]のセキュリティ認証情報タブから設定できますが、具体的な設定手順の流れはルートユーザー MFA 認証の設定と同じなので割愛します。

![IAMユーザーのMFA設定](/images/f8705b6cf2e2b9/image26.png =500x)

②MFA設定が完了しました。
![MFA設定完了](/images/f8705b6cf2e2b9/image27.png =500x)
## 6. Cost Explorerの有効化

Cost Explorerを有効化しないと、AWSアカウントの利用日に関する情報にアクセスできないため、早めに有効化しておきましょう。

①Billing and Cost Management(請求とコスト管理)のコンソールへ移動し、コスト分析 > Cost Explorerにアクセスすると、以下のようなアナウンス文が表示されました。

:::message
これは初めての訪問であるため、コストと使用状況のデータを準備するには時間がかかります。24 時間後にもう一度確認してください。
:::

②案内に従って24時間後に再アクセスします。請求とコスト管理のホーム画面が表示されていれば有効化完了です。
![Cost Explorer有効化完了](/images/f8705b6cf2e2b9/image28.png =500x)
