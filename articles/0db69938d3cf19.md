# はじめに

この資料は、AWSをこれから始める社内メンバー向けの入門ガイドです。
AWSの概要と基本的な特徴を理解することを目的としています。

# AWSとは

AWSは「Amazon Web Services」の略称で、Amazonが提供するクラウドコンピューティングサービスです。
サーバーやストレージ、データベースなど、ITインフラに必要な様々なサービスをインターネット経由で利用できます。

# オンプレミスとの違い

## オンプレミスの場合
- 自社でサーバールームを用意
- 物理的なサーバー機器を購入
- 機器の設置・メンテナンスを自社で実施
- 電源・空調の管理が必要

## AWSの場合
- サーバールーム不要
- 物理的な機器の購入不要
- メンテナンスはAWSが実施
- 必要な分だけ利用可能

# メリット
![AWSのメリット](/images/0db69938d3cf19/image01.png =600x)

1. **初期投資を抑えられる**
   - サーバー機器の購入が不要
   - データセンターの構築費用が不要

2. **柔軟なリソース調整**
   - 必要に応じてサーバーの性能を上げ下げ可能
   - 利用していない時間帯は停止して費用節約も可能

3. **運用負荷の軽減**
   - ハードウェアの管理が不要
   - セキュリティアップデートの多くをAWSが対応

# デメリット

1. **コスト管理の難しさ**
   - 利用料金の予測が難しい
   - 使い方を誤ると予想以上の請求額に

2. **学習コスト**
   - 多数のサービスがあり、使いこなすまでに時間が必要
   - 設定ミスによるトラブルのリスク

3. **ネットワーク依存**
   - インターネット接続が必須
   - 通信速度がサービスの性能に影響

# グローバルインフラストラクチャ

AWSは世界中にデータセンターを持っています：
![AWSのグローバルインフラストラクチャ](/images/0db69938d3cf19/image02.png =600x)

- **リージョン**
  - 東京、大阪などの地理的な拠点
  - 各リージョンは独立して運用

- **アベイラビリティーゾーン（AZ）**
  - 各リージョン内の独立したデータセンター
  - 災害対策のため物理的に離れた場所に設置


# AWSで作れるもの

## Webサービス関連
- **Webサイト・アプリケーション**
  - 企業のホームページ
  - ECサイト
  - スマートフォンアプリのバックエンド
  - 社内業務システム

## データ処理関連
- **データ分析基盤**
  - ビッグデータの収集・分析
  - 機械学習の実行環境
  - ログ分析システム

## ストレージ関連
- **各種データの保管**
  - バックアップシステム
  - 社内文書管理システム
  - 動画・画像の配信システム

## その他
- **IoTシステム**
  - センサーデータの収集・分析
  - スマートデバイスの管理システム

- **開発環境**
  - CI/CD（継続的な開発・デプロイ）環境
  - テスト環境
  - ステージング環境
