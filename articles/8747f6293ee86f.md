この資料は、AWSをこれから始める社内メンバー向けの入門ガイドです。

# IAMとは

IAM（Identity and Access Management）は、AWSのサービスやリソースへのアクセスを安全に管理するためのサービスです。以下の主要な機能を提供します：

- AWSリソースへのアクセス制御
- 詳細な権限管理
- 一時的な認証情報の発行
- 無料で利用可能

## IAMの主要な構成要素

### 1. IAMユーザー
- AWSを利用する個人や、アプリケーションの個別のアカウント
- 固有の認証情報（ユーザー名/パスワード、アクセスキー）を持つ
- 直接またはグループを通じて権限を付与可能

### 2. IAMグループ
- IAMユーザーの集合
- 複数のユーザーに対して一括で権限管理が可能
- ユーザーは複数のグループに所属可能

### 3. IAMポリシー
- JSONドキュメント形式で記述された権限の定義
- 許可または拒否の権限を詳細に指定可能
- AWS管理ポリシーとカスタマー管理ポリシーの2種類

### 4. IAMロール
- 一時的な権限を提供するための仕組み
- EC2インスタンスやLambda関数などのAWSサービスに付与
- 他のAWSアカウントのIAMユーザーにも付与可能
- ロールはポリシーをまとめたもの

# IAMの基本操作

## 1. IAMユーザーの管理

### ユーザーの作成
1. IAMコンソールから「ユーザー」→「ユーザーを作成」を選択
![IAMユーザー作成](/images/8747f6293ee86f/image01.png =500x)

2. ユーザー情報を入力
    - ユーザー名はユーザーの識別に使用されるため、一意である必要があります。
    - ユーザーのタイプを選択（IAM or Iam Identity Center）
    - パスワードを自動生成する場合は「パスワードを生成」を選択します。
    - 次回ログイン時にパスワードを変更する場合はチェックを入れます。
:::message alert
SSOを使用している場合は、ユーザーのタイプを「Iam Identity Center」にします。
:::
![IAMユーザー名入力](/images/8747f6293ee86f/image02.png =500x)

3. 権限の設定（直接付与・グループに追加・既存ユーザーをコピー）
    - グループに所属させる場合は「グループに追加」を選択します。
    - 既存ユーザーをコピーする場合は「既存ユーザーをコピー」を選択します。
    - ユーザーに直接付与する場合は「直接付与」を選択します。
![IAMユーザー権限設定](/images/8747f6293ee86f/image03.png =500x)

4. 確認画面で「ユーザーの作成」を選択
![IAMユーザー確認](/images/8747f6293ee86f/image04.png =500x)

### ユーザーの削除
1. IAMコンソールから対象ユーザーを選択
![IAMユーザー選択](/images/8747f6293ee86f/image06.png =500x)

2. 「削除」を実行
![IAMユーザー削除](/images/8747f6293ee86f/image07.png =500x)
:::message alert
  - ユーザーを削除する前に、以下の項目を必ず確認してください：
  - ユーザーに割り当てられている権限
  - 関連付けられているリソース
  - アクティブなアクセスキー
  - MFAデバイスの設定状況
:::


### パスワードのリセット
1. IAMコンソールから対象ユーザーを選択
![IAMユーザー選択](/images/8747f6293ee86f/image06.png =500x)

2. 「セキュリティ認証情報」タブで「コンソールアクセスの管理」
![IAMユーザーパスワード管理](/images/8747f6293ee86f/image08.png =500x)
![IAMユーザーパスワードリセット](/images/8747f6293ee86f/image09.png =500x)

3. 新しいパスワードを設定または自動生成
![IAMユーザーパスワードリセット](/images/8747f6293ee86f/image10.png =500x)

4. パスワードをリセットを選択

## 2. MFAの設定

### MFAの有効化
1. IAMコンソールでユーザー選択→「セキュリティ認証情報」
![IAMユーザー選択](/images/8747f6293ee86f/image06.png =500x)
![IAMユーザーMFA有効化](/images/8747f6293ee86f/image14.png =500x)

2.「MFAデバイスの割り当て」を選択
![IAMユーザーMFAデバイス割り当て](/images/8747f6293ee86f/image14.png =500x)

3. デバイス名とデバイスタイプを入力
![IAMユーザーMFAデバイス割り当て](/images/8747f6293ee86f/image11.png =500x)

4. デバイス認証用コードを取得
![IAMユーザーMFAデバイスタイプ選択](/images/8747f6293ee86f/image12.png =500x)

5. デバイスが登録される
![IAMユーザーMFAデバイス設定](/images/8747f6293ee86f/image13.png =500x)

### MFAデバイス削除
1. IAMコンソールでユーザー選択→「セキュリティ認証情報」
![IAMユーザー選択](/images/8747f6293ee86f/image06.png =500x)
![IAMユーザーMFA有効化](/images/8747f6293ee86f/image14.png =500x)

2. 「MFAデバイス」の削除を選択
![MFAデバイス削除](/images/8747f6293ee86f/image15.png =500x)

3. 対象を確認後、削除を実行
![MFAデバイス削除](/images/8747f6293ee86f/image16.png =500x)

## 3. IAMグループの管理

### グループの作成
1. IAMコンソールから「ユーザーグループ」→「グループの作成」
2. グループ名を設定
3. ポリシーをアタッチ
4. メンバーを追加（オプション）
5. ユーザーグループの作成を選択
![IAMグループ作成](/images/8747f6293ee86f/image17.png =500x)
